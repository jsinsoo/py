Flask

01. Flask 기본 웹 서버 만들기

가장 기본적인 Flask 웹 서버입니다. 브라우저에서 / 주소로 접속하면 Hello Flask! 문자열이 출력됩니다.

from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello Flask!"

if __name__ == "__main__":
    app.run(debug=True)

02. Flask HTML 문자열 반환

Flask 함수는 단순 문자열뿐 아니라 HTML 문자열도 반환할 수 있습니다. 간단한 웹페이지를 빠르게 만들 때 유용합니다.

03. Flask URL 변수 사용

URL 경로에 들어온 값을 함수의 매개변수로 받아 동적인 페이지를 만들 수 있습니다. /user/Kim으로 접속하면 Kim님 환영합니다!가 표시됩니다.

from flask import Flask

app = Flask(__name__)

@app.route("/user/<name>")
def user(name):
    return f"<h1>{name}님 환영합니다!</h1>"

if __name__ == "__main__":
    app.run(debug=True)

04. Flask GET 요청 파라미터 처리

주소 뒤의 ?q=python 같은 쿼리 파라미터를 request.args.get()으로 읽어옵니다. 검색 기능의 기본 구조입니다.

from flask import Flask, request

app = Flask(__name__)

@app.route("/search")
def search():
    keyword = request.args.get("q", "검색어 없음")
    return f"<h1>검색어: {keyword}</h1>"

if __name__ == "__main__":
    app.run(debug=True)

05. Flask POST 폼 데이터 처리

GET일 때는 입력 폼을 보여주고, POST일 때는 사용자가 입력한 name 값을 받아 처리합니다. 로그인, 설문, 제출 기능의 기초입니다.

from flask import Flask, request

app = Flask(__name__)

@app.route("/", methods=["GET", "POST"])
def form():
    if request.method == "POST":
        name = request.form.get("name")
        return f"<h1>{name}님 제출 완료!</h1>"

    return """
    <form method="post">
        <input name="name" placeholder="이름 입력">
        <button type="submit">제출</button>
    </form>
    """

if __name__ == "__main__":
    app.run(debug=True)

상세 설명

  • 핵심 목적: 웹 브라우저의 요청을 받아 서버가 응답을 돌려주는 구조를 이해합니다.
  • 중요 개념: URL 라우팅, view 함수, 요청 객체, 응답 객체, HTML 또는 JSON 반환입니다.

07. Django 기본 view 함수

Django에서 view 함수는 요청을 받아 응답을 반환합니다. urls.py에서 주소와 view 함수를 연결해야 브라우저에서 접근할 수 있습니다.

# views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello Django!")

# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path("", views.home, name="home"),
]

08. Django HTML 템플릿 렌더링

Django의 render()는 HTML 템플릿과 데이터를 결합해 웹페이지를 만듭니다. {{ title }} 같은 템플릿 변수가 context 값으로 채워집니다.

# views.py
from django.shortcuts import render

def home(request):
    context = {
        "title": "Django 수업",
        "message": "템플릿으로 HTML을 렌더링합니다."
    }
    return render(request, "home.html", context)

# templates/home.html
<h1>{{ title }}</h1>
<p>{{ message }}</p>

상세 설명

  • 핵심 목적: 웹 브라우저의 요청을 받아 서버가 응답을 돌려주는 구조를 이해합니다.

09. Django URL 변수 사용

Django에서도 URL 일부를 변수로 받을 수 있습니다. profile/kim/으로 접속하면 username 값으로 kim이 전달됩니다.

# views.py
from django.http import HttpResponse

def profile(request, username):
    return HttpResponse(f"<h1>{username}의 프로필</h1>")

# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path("profile/<str:username>/", views.profile, name="profile"),
]

10. Django 간단한 게시글 목록 페이지

Django 템플릿의 for 반복문을 사용해 게시글 목록을 화면에 출력하는 예제입니다. 실제 게시판 기능의 기초 구조입니다.

# views.py
from django.shortcuts import render

def post_list(request):
    posts = [
        {"title": "첫 번째 글", "author": "Kim"},
        {"title": "두 번째 글", "author": "Lee"},
        {"title": "세 번째 글", "author": "Park"},
    ]
    return render(request, "post_list.html", {"posts": posts})

# templates/post_list.html
<h1>게시글 목록</h1>
<ul>
{% for post in posts %}
    <li>{{ post.title }} - {{ post.author }}</li>
{% endfor %}
</ul>

상세 설명

  • 핵심 목적: 웹 브라우저의 요청을 받아 서버가 응답을 돌려주는 구조를 이해합니다.
  • 중요 개념: URL 라우팅, view 함수, 요청 객체, 응답 객체, HTML 또는 JSON 반환입니다.
  • 확장 방향: 입력 폼, 데이터베이스, 공공데이터 API, 로그인, 대시보드, AI 모델 연결로 확장할 수 있습니다.
