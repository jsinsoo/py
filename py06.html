🧠 핵심 개념

1. 클래스와 객체

클래스는 객체를 만들기 위한 설계도이고, 객체는 클래스로부터 실제로 만들어진 대상입니다.

2. 생성자와 self

__init__은 객체가 생성될 때 자동 실행되는 생성자입니다. self는 현재 객체 자신을 가리킵니다.

3. 메서드

메서드는 클래스 안에 정의된 함수입니다. 객체가 수행할 수 있는 행동을 표현합니다.

4. 상속

상속은 기존 클래스의 속성과 기능을 물려받아 새로운 클래스를 만드는 방법입니다.

5. 다형성

다형성은 같은 이름의 메서드가 객체에 따라 다르게 동작하는 성질입니다.

6. 캡슐화

캡슐화는 데이터를 직접 건드리지 못하게 보호하고, 정해진 메서드를 통해 접근하도록 하는 설계 방식입니다.

💻 개념기반 학습코드 10개 + 상세 설명

코드 1. 클래스와 객체

class Student:
    pass

s1 = Student()
print(type(s1).__name__)

Student는 클래스이고, s1은 Student 클래스로 만든 객체입니다.

코드 2. 생성자 __init__

class Student:
    def __init__(self, name):
        self.name = name

s = Student("민수")
print(s.name)

객체가 생성될 때 __init__이 자동 실행되고, name 값이 객체에 저장됩니다.

코드 3. 메서드

class Student:
    def __init__(self, name):
        self.name = name
    def introduce(self):
        print("저는", self.name, "입니다.")

s = Student("지아")
s.introduce()

introduce()는 객체가 자기 이름을 출력하는 메서드입니다.

코드 4. 은행 계좌 입금

class Account:
    def __init__(self, balance):
        self.balance = balance
    def deposit(self, money):
        self.balance += money

a = Account(1000)
a.deposit(500)
print(a.balance)

객체의 balance 속성에 입금액을 더해 잔액을 변경합니다.

코드 5. 캡슐화

class Account:
    def __init__(self):
        self.__balance = 0
    def deposit(self, money):
        if money > 0:
            self.__balance += money
    def get_balance(self):
        return self.__balance

a = Account()
a.deposit(1000)
print(a.get_balance())

__balance는 외부에서 직접 접근하기 어렵게 만든 속성입니다.

코드 6. 클래스 변수

class Student:
    school = "Python High"
    def __init__(self, name):
        self.name = name

s1 = Student("A")
s2 = Student("B")
print(s1.school, s2.school)

school은 모든 객체가 공유하는 클래스 변수입니다.

코드 7. 상속

class Animal:
    def speak(self):
        print("소리를 냅니다.")
class Dog(Animal):
    pass

d = Dog()
d.speak()

Dog 클래스는 Animal 클래스를 상속받아 speak() 메서드를 사용할 수 있습니다.

코드 8. 다형성

class Cat:
    def speak(self):
        print("야옹")
class Dog:
    def speak(self):
        print("멍멍")

for animal in [Cat(), Dog()]:
    animal.speak()

같은 speak() 호출이 객체 종류에 따라 다르게 실행됩니다.

코드 9. staticmethod

class MathTool:
    @staticmethod
    def add(a, b):
        return a + b

print(MathTool.add(3, 4))

staticmethod는 객체 상태를 사용하지 않는 기능을 클래스 안에 묶을 때 사용합니다.

코드 10. classmethod

class Student:
    count = 0
    def __init__(self):
        Student.count += 1
    @classmethod
    def show_count(cls):
        print(cls.count)

s1 = Student()
s2 = Student()
Student.show_count()

classmethod는 cls를 통해 클래스 변수에 접근할 수 있습니다.

🧪 실습 예제

1. 은행 계좌 시스템

class Account:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance
    def deposit(self, money):
        if money > 0:
            self.__balance += money
    def withdraw(self, money):
        if 0 < money <= self.__balance:
            self.__balance -= money
    def info(self):
        print(self.owner, self.__balance)

a = Account("Kim", 10000)
a.deposit(5000)
a.withdraw(3000)
a.info()

2. 학생 관리 시스템

class Student:
    def __init__(self, name, scores):
        self.name = name
        self.scores = scores
    def average(self):
        return sum(self.scores) / len(self.scores)

s = Student("Lee", [90, 80, 100])
print(s.name, s.average())

3. 게임 캐릭터 클래스

class Character:
    def __init__(self, name, hp, power):
        self.name = name
        self.hp = hp
        self.power = power
    def attack(self, target):
        target.hp -= self.power
        print(self.name, "공격!", target.name, "남은 HP:", target.hp)

hero = Character("용사", 100, 20)
monster = Character("슬라임", 50, 10)
hero.attack(monster)

🌟 수업마무리하기

객체지향 프로그래밍은 프로그램을 현실 세계의 대상처럼 구조화하는 사고방식입니다.

  • 클래스는 설계도이다.
  • 객체는 설계도로 만든 실제 대상이다.
  • 생성자는 객체 초기 상태를 만든다.
  • 메서드는 객체의 행동이다.
  • 상속은 코드 재사용을 돕는다.
  • 다형성은 객체별 다른 실행을 가능하게 한다.
  • 캡슐화는 데이터를 안전하게 보호한다.

핵심 정리: OOP는 대규모 프로그램을 객체 단위로 나누어 관리하는 설계 방법입니다.
