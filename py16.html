01. Flask 기본 웹 서버 만들기

from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello Flask!"

if __name__ == "__main__":
    app.run(debug=True)

✍️ 출력 결과 서술형

/ 주소로 접속했을 때 브라우저에 보이는 응답 문자열을 쓰시오.

정답
Hello Flask!
상세 풀이

@app.route('/')가 기본 주소와 home 함수를 연결하고, home 함수가 Hello Flask!를 반환합니다.

⭕❌ 개념 OX 퀴즈

@app.route('/')는 Flask에서 URL과 함수를 연결하는 라우팅 역할을 한다.

정답: O
상세 풀이: route 데코레이터는 특정 URL 요청이 들어왔을 때 실행할 함수를 지정합니다.

03. Flask URL 변수 사용

from flask import Flask

app = Flask(__name__)

@app.route("/user/<name>")
def user(name):
    return f"<h1>{name}님 환영합니다!</h1>"

if __name__ == "__main__":
    app.run(debug=True)

✍️ 출력 결과 서술형

정답
Kim님 환영합니다!
상세 풀이

URL의 <name> 자리에 Kim이 들어가 user 함수의 name 매개변수로 전달됩니다.

⭕❌ 개념 OX 퀴즈

Flask의 /user/<name> 구조는 URL 일부를 변수처럼 받을 수 있다.

정답: O
상세 풀이: 꺾쇠 괄호로 표시한 URL 조각은 함수 매개변수로 전달됩니다.

04. Flask GET 요청 파라미터 처리

from flask import Flask, request

app = Flask(__name__)

@app.route("/search")
def search():
    keyword = request.args.get("q", "검색어 없음")
    return f"<h1>검색어: {keyword}</h1>"

if __name__ == "__main__":
    app.run(debug=True)

✍️ 출력 결과 서술형

/search?q=python 주소로 접속했을 때 h1에 표시되는 문구를 쓰시오.

정답
검색어: python
상세 풀이

request.args.get('q')가 URL 쿼리 파라미터 q의 값 python을 읽습니다.

⭕❌ 개념 OX 퀴즈

request.args.get('q')는 GET 쿼리 파라미터 q 값을 읽는 데 사용할 수 있다.

정답: O
상세 풀이: request.args는 URL 뒤의 ?q=python 같은 쿼리 문자열을 다룹니다.

🧩 심화 5지선다형

주소 /search?q=ai에서 q 값을 읽는 코드는?

정답: 1번
상세 풀이: GET 쿼리 파라미터는 request.args에서 가져옵니다.

05. Flask POST 폼 데이터 처리

from flask import Flask, request

app = Flask(__name__)

@app.route("/", methods=["GET", "POST"])
def form():
    if request.method == "POST":
        name = request.form.get("name")
        return f"<h1>{name}님 제출 완료!</h1>"

    return """
    <form method="post">
        <input name="name" placeholder="이름 입력">
        <button type="submit">제출</button>
    </form>
    """

if __name__ == "__main__":
    app.run(debug=True)

✍️ 출력 결과 서술형

폼에 name 값으로 민수를 입력하고 POST 제출했을 때 표시되는 문구를 쓰시오.

정답
민수님 제출 완료!
상세 풀이

POST 요청이면 request.form.get('name')으로 입력값 민수를 가져와 응답에 넣습니다.

⭕❌ 개념 OX 퀴즈

Flask에서 POST 폼 입력값은 request.form을 통해 읽을 수 있다.

정답: O
상세 풀이: HTML form의 method가 post이면 request.form으로 전송 데이터를 읽습니다.

06. Flask JSON API 서버 만들기

from flask import Flask, jsonify

app = Flask(__name__)

@app.route("/api/weather")
def weather_api():
    data = {
        "city": "Jeonju",
        "temp": 25,
        "weather": "맑음"
    }
    return jsonify(data)

if __name__ == "__main__":
    app.run(debug=True)

✍️ 출력 결과 서술형

/api/weather 요청 시 JSON 데이터의 city 값은 무엇인지 쓰시오.

정답
Jeonju
상세 풀이

data 딕셔너리에서 city 키의 값은 Jeonju이며 jsonify가 JSON으로 변환합니다.

⭕❌ 개념 OX 퀴즈

jsonify(data)는 딕셔너리를 JSON 응답으로 변환하는 데 사용할 수 있다.

정답: O
상세 풀이: jsonify는 딕셔너리나 리스트를 JSON HTTP 응답으로 만들어 줍니다.

08. Django HTML 템플릿 렌더링

# views.py
from django.shortcuts import render

def home(request):
    context = {
        "title": "Django 수업",
        "message": "템플릿으로 HTML을 렌더링합니다."
    }
    return render(request, "home.html", context)

# templates/home.html
<h1>{{ title }}</h1>
<p>{{ message }}</p>

✍️ 출력 결과 서술형

템플릿의 {{ title }} 위치에 렌더링되는 문구를 쓰시오.

정답
Django 수업
상세 풀이

context의 title 키 값이 템플릿의 {{ title }} 자리에 들어갑니다.

⭕❌ 개념 OX 퀴즈

render(request, 'home.html', context)는 템플릿과 데이터를 결합해 HTML 응답을 만들 수 있다.

정답: O
상세 풀이: render는 템플릿에 context 데이터를 적용해 최종 HTML을 반환합니다.

09. Django URL 변수 사용

# views.py
from django.http import HttpResponse

def profile(request, username):
    return HttpResponse(f"<h1>{username}의 프로필</h1>")

# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path("profile/<str:username>/", views.profile, name="profile"),

✍️ 출력 결과 서술형

정답

상세 풀이

URL의 <str:username>에 kim이 들어가 profile 함수의 username으로 전달됩니다.

⭕❌ 개념 OX 퀴즈

Django의 path('profile/<str:username>/', ...) 구조는 URL 일부를 문자열 변수로 받을 수 있다.

정답: O
상세 풀이: Django path converter는 URL 일부를 변수로 view에 전달합니다.

10. Django 간단한 게시글 목록 페이지

# views.py
from django.shortcuts import render

def post_list(request):
    posts = [
        {"title": "첫 번째 글", "author": "Kim"},
        {"title": "두 번째 글", "author": "Lee"},
        {"title": "세 번째 글", "author": "Park"},
    ]
    return render(request, "post_list.html", {"posts": posts})

# templates/post_list.html
<h1>게시글 목록</h1>
<ul>
{% for post in posts %}
    <li>{{ post.title }} - {{ post.author }}</li>
{% endfor %}
</ul>

✍️ 출력 결과 서술형

첫 번째 게시글이 템플릿의 li에 표시되는 형태를 쓰시오.

정답
첫 번째 글 - Kim
상세 풀이

첫 번째 post의 title은 첫 번째 글, author는 Kim이며 템플릿에서 둘을 하이픈으로 연결합니다.

⭕❌ 개념 OX 퀴즈

Django 템플릿에서 {% for post in posts %}는 posts 목록을 반복 출력할 때 사용할 수 있다.

정답: O
상세 풀이: Django 템플릿 for문은 리스트 데이터를 반복 렌더링합니다.
